# event_detail.csv 数据导入说明

## 一、数据来源

- 文件：`~/data/event_detail.csv`
- 记录数：1000 条
- 内容：与石油价格相关的新闻事件，包含事件类型、情绪方向、情绪强度、新闻标题、摘要、来源和链接。

原始 CSV 字段说明：

| 字段名 | 类型 | 含义 |
|---|---|---|
| event_id | string | CSV 内部编号（如 event_1），仅用于标识，不入库 |
| event_time | datetime | 事件/新闻发布时间 |
| event_type | string | 事件类型：policy（政策）/ geo（地缘）/ supply（供给） |
| sentiment_direction | int | 情绪方向：+1（利多）/ -1（利空） |
| sentiment_intensity | float | 情绪强度绝对值，范围 0～1 |
| title | string | 新闻标题（英文） |
| summary | string | 新闻对油价影响的中文摘要 |
| source | string | 新闻来源（如"路透社 (Reuters)"） |
| url | string | 原文链接 |

## 三、存储方案：一行 CSV → 三张表

每一行 CSV 数据对应**三条数据库记录**，分别写入以下三张已有的表：

```
event_detail.csv 的一行
        │
        ├──► news_article     （新闻原文信息）
        ├──► extreme_event    （事件分类与强度）
        └──► sentiment_score  （量化情绪分值）
```

三张表通过 `news_id` 关联，`extreme_event.news_id` 和 `sentiment_score.news_id` 均指向 `news_article.id`。

## 四、字段映射详情

### 1. `news_article` 表

| 数据库字段 | 来源 | 说明 |
|---|---|---|
| `source` | `source` | 新闻来源机构 |
| `title` | `title` | 新闻标题 |
| `content` | — | 写入 `NULL`，CSV 中无正文全文 |
| `url` | `url` | 原文链接 |
| `published_at` | `event_time` | 新闻发布时间 |
| `ingested_at` | 脚本运行时间 | 数据入库时间，取当前 UTC 时间 |

### 2. `extreme_event` 表

| 数据库字段 | 来源 | 说明 |
|---|---|---|
| `news_id` | `news_article.id` | 关联刚插入的新闻记录 |
| `event_type` | `event_type` | 事件类型（policy / geo / supply） |
| `summary` | `summary` | 对油价影响的中文摘要 |
| `intensity` | `sentiment_direction × sentiment_intensity` | 带符号的情绪强度，正值=利多，负值=利空 |
| `occurred_at` | `event_time` | 事件发生时间 |
| `created_at` | 脚本运行时间 | 记录创建时间，取当前 UTC 时间 |

### 3. `sentiment_score` 表

| 数据库字段 | 来源 | 说明 |
|---|---|---|
| `news_id` | `news_article.id` | 关联刚插入的新闻记录 |
| `sentiment` | `sentiment_direction × sentiment_intensity` | 带符号的情绪分，与 `extreme_event.intensity` 相同 |
| `confidence` | — | 写入 `NULL`，CSV 中无置信度数据 |
| `scored_at` | `event_time` | 情绪评分时间 |

> **`intensity` / `sentiment` 的计算方式**：
> `signed_score = sentiment_direction × |sentiment_intensity|`
> 例：direction=-1，intensity=0.75 → 入库值 = **-0.75**（负值表示看空）

## 五、导入脚本

脚本位置：`~/data/import_events.py`

执行方式：
```bash
python3 ~/data/import_events.py
```

脚本逻辑（伪代码）：
```
打开 CSV 文件
对每一行：
    1. 计算 signed_score = direction × |intensity|
    2. INSERT INTO news_article → 获得 news_id
    3. INSERT INTO extreme_event（绑定 news_id）
    4. INSERT INTO sentiment_score（绑定 news_id）
提交事务
```

## 六、导入结果验证

导入完成后，执行以下 SQL 可验证数据：

```sql
-- 各表记录数
SELECT COUNT(*) FROM news_article;      -- 应含导入的 1000 条
SELECT COUNT(*) FROM extreme_event;     -- 1000 条
SELECT COUNT(*) FROM sentiment_score;   -- 1000 条

-- 联表查看一条完整记录
SELECT na.title, na.source, ee.event_type, ee.intensity, ss.sentiment, na.published_at
FROM news_article na
JOIN extreme_event ee ON ee.news_id = na.id
JOIN sentiment_score ss ON ss.news_id = na.id
LIMIT 5;
```
